\section{Simulation Results}
\label{sec:simulation_results}

\subsection{Reporting Conventions and Interpretation Boundaries}
\label{subsec:results_reporting_conventions}
Chapter~8 reports DTSE outputs for the stress scenarios in Section~\ref{sec:stress_scenarios}. Results are descriptive and comparative: each scenario is evaluated as a deviation from the corresponding neutral baseline trajectory using the pre-specified metric set (Section~\ref{sec:evaluation_metrics}). The chapter operationalizes the thesis comparison by applying matched stress inputs to the ONO profile and comparator DePIN mechanism profiles.

Profile IDs, the scenario grid, and run configuration are fixed in the frozen run manifest (Appendix Section~\ref{app:dtse_run_manifest}). Comparability is enforced by holding the reporting horizon, scenario inputs, and aggregation conventions constant across profiles. Stochasticity is handled through repeated runs under a fixed-seed policy; outcomes are reported with distributional summaries (e.g., medians and interquartile ranges) rather than single trajectories. Figures use consistent axes and scenario labels to support direct visual comparison.

Metric-applicability markings from the empirical layer (including \emph{N/R} in retrospective windows) do not constrain DTSE reporting here: this chapter reports the full pre-specified metric family under frozen simulation inputs and controlled assumptions.

Baseline anchoring follows output class. ONO timing tables report threshold crossings relative to the ONO baseline. Cross-profile figures report each profile as a deviation from its own baseline under the same frozen scenario inputs.

Two interpretation constraints apply throughout:
\begin{enumerate}
    \item Baseline trajectories can drift over the 52-week neutral horizon; stress is therefore interpreted as deviation from a drifting reference path rather than from a stable-equilibrium baseline.
    \item Selected metric families can sit on materially different numeric scales across profiles in the same scenario; cross-profile comparisons are therefore directional and scenario-conditional rather than magnitude-equivalent rankings.
\end{enumerate}

Broader governance interpretation and thesis-level implications are taken up in Section~\ref{sec:discussion}. This chapter restricts itself to three tasks: (i) establish the neutral reference path, (ii) report scenario outcomes as deviations from that path, and (iii) formalize the failure-signature vocabulary used later in the discussion chapter.

\subsection{Baseline Behavior (Neutral Reference Runs)}
\label{baseline-behavior-sanity-check}
Baseline runs establish the neutral reference trajectories for core DTSE state variables (e.g., modeled price signal, token supply and emissions/burns, provider participation, utilization, and incentive-solvency proxies) in the absence of scenario-driven stress inputs. They serve as comparison anchors for Chapter~8 and are not interpreted as equilibrium states.

All baseline experiments use the standard DTSE configuration defined in Section~\ref{sec:methodology}. Scenario logic is disabled and exogenous inputs are set to neutral conditions so that baseline dynamics arise from the interaction between each mechanism profile's rule set and modeled provider economics.

Under neutral conditions, aggregate state variables typically evolve smoothly due to deterministic rule components and constrained exogenous inputs. Provider-level metrics (e.g., profit and provider churn) can retain dispersion because provider heterogeneity and stochastic decision components remain active even when macro and demand inputs are held fixed. Provider retention is reported only for profiles with non-zero initial active provider counts; where initial active provider counts are zero, provider retention is treated as undefined rather than imputed.

Baseline drift matters for interpretation. In DTSE, the baseline is the neutral reference path generated by the frozen rule set under neutral inputs; it is not a claim of stability. Stress results in this chapter are therefore compared against that reference path rather than against a flat line. When the baseline already trends (as in the ONO baseline, where modeled price and active provider count decline over the horizon), stress is read as an acceleration, reversal, or earlier threshold crossing relative to the reference. The key questions are when a scenario pulls the system away from baseline, how early the departure appears, and which metric family registers it first.

Demand contraction illustrates how Chapter~8 should be read. In that scenario, ONO first registers stress through utilization (week~12), while active provider count effects emerge materially later. The result is therefore not a binary ``stable versus unstable'' classification. The result is a response pattern: under a given stress channel, which subsystem moves first, when the departure from baseline becomes detectable, and how the deviation propagates through the modeled network.

\subsubsection{Demand contraction}
Demand contraction isolates Reward--Demand Decoupling: usage weakens while reward logic remains active. The diagnostic question is which subsystem registers first under baseline-relative detection, and whether participation effects appear only after profitability and provider churn thresholds begin to bind.

In the frozen scenario grid, demand contraction applies a high-to-decay demand regime with a lower demand baseline (10{,}000 versus 12{,}000 in baseline), 8\% demand volatility, and a bearish macro modifier (Appendix Section~\ref{app:dtse_scenario_grid}). Macro is treated as part of the channel definition and held fixed for this scenario. Signal timing follows the baseline-relative utilization-deviation rule (10\%) in Appendix Section~\ref{app:dtse_threshold_table}.

Under the demand-regime inputs and update rule (Appendix Sections~\ref{app:dtse_scenario_grid} and~\ref{app:dtse_scenario_formula_map}), week-12 demand is approximately 10{,}820 under contraction versus 12{,}000 under baseline, i.e., approximately a 10\% gap.

In ONO, utilization and usage-linked sinks diverge before active provider count effects become dominant (Figure~\ref{fig:dtse_demand_contraction_ono}); first utilization deviation appears at week 12 and active provider count degradation at week 49 (Table~\ref{tab:dtse_signal_timing_ono}).

\subsubsection{Liquidity shock}
Liquidity shock isolates the transmission of a discrete token-liquidity dislocation into provider-side stress fields.

In the scenario configuration, the liquidity event occurs at week 20 with a 35\% unlock-sell scalar and initial liquidity of 250{,}000 under neutral demand and macro settings (Appendix Sections~\ref{app:dtse_scenario_grid} and~\ref{app:dtse_threshold_table}). Price and participation timing are read against baseline using the frozen 10\% price-deviation rule (Appendix Section~\ref{app:dtse_threshold_table}).

Under the finite-depth liquidity rule, the 35\% reserve-depth sell event produces an abrupt modeled price dislocation with realized step size determined by the liquidity-shock rule and run-state conditions (Appendix Sections~\ref{app:dtse_scenario_formula_map} and~\ref{app:dtse_threshold_table}).

In ONO, a discrete modeled dislocation appears around the event week (Figure~\ref{fig:dtse_liquidity_shock_ono}); the price signal field crosses at week 21, while the lagging detection field records provider churn at week 20 (Table~\ref{tab:dtse_signal_timing_ono}). The lagging detection field is a diagnostic label and is not a guaranteed later timestamp than the earliest-trigger field.

\subsubsection{Competitive yield pressure}
Competitive yield pressure isolates participation sensitivity to outside opportunities without requiring demand collapse or a discrete liquidity event.

In the scenario configuration, the external-yield scalar is set to 1.5 under neutral demand and macro controls (Appendix Sections~\ref{app:dtse_scenario_grid} and~\ref{app:dtse_threshold_table}). The provider-switching block applies higher pressure to \emph{Provider Type A} and lower pressure to \emph{Provider Type B} with bounded random variation; in the ONO calibration, Provider Type A/B maps to rural/urban operator classes.

Baseline-relative detection uses a 5\% active provider count drop rule and a 20\% provider churn increase rule (Appendix Section~\ref{app:dtse_threshold_table}). At an external-yield scalar of 1.5, modeled base switching pressure is about 18.4\%; before random scaling, this corresponds to approximately 27.6\% weekly pressure for Provider Type A and 9.2\% for Provider Type B, as implied by the switching-response rule definitions (Appendix Sections~\ref{app:dtse_scenario_formula_map} and~\ref{app:dtse_threshold_table}).

In ONO, this channel registers primarily in participation metrics rather than abrupt capacity discontinuities (Figure~\ref{fig:dtse_competitive_yield_ono}); provider churn deviation appears at week 1 and active provider count deviation at week 0 (Table~\ref{tab:dtse_signal_timing_ono}).

\subsubsection{Provider cost inflation}
Provider cost inflation tests Subsidy-Gap exposure through an exogenous operating-cost increase.

In the scenario configuration, baseline provider cost is multiplied by 1.35 under neutral demand and macro settings (Appendix Sections~\ref{app:dtse_scenario_grid} and~\ref{app:dtse_threshold_table}), and the change is propagated through heterogeneous cost multipliers and configured dispersion.

\noindent\emph{Note:} \emph{Provider Type A/B} denotes switching-heterogeneity classes used in the competitive-yield mechanism, while \emph{Cost Tier 1/2} denotes cost segmentation used only in the provider-cost inflation mechanism.

Profit and participation fields are evaluated with the frozen baseline-relative rules (10\% profit deviation and 5\% active provider count deviation). Because provider cost rises immediately at week 0, profit is expected to move before participation. For example, baseline \$12 becomes \$16.2; a lower-cost Cost Tier 1 profile yields approximately \$12.96, while higher-cost Cost Tier 2 profiles remain substantially higher before noise, consistent with the provider-cost multiplier structure (Appendix Sections~\ref{app:dtse_scenario_formula_map} and~\ref{app:dtse_threshold_table}).

In ONO, cost inflation compresses margins first, then participation follows (Figure~\ref{fig:dtse_cost_inflation_ono}); the first profit signal appears at week 0 and the lagging detection field (active provider count) is detected at week 2 (Table~\ref{tab:dtse_signal_timing_ono}).

\subsubsection{Cross-Profile and Cross-Scenario Synthesis}
Across the experiment set, stress channels manifest in distinct metric families and timing windows (Figures~\ref{fig:dtse_cross_profile_retention}--\ref{fig:dtse_signal_timing_chart}, Tables~\ref{tab:dtse_signal_timing_ono}--\ref{tab:dtse_cross_scenario_synthesis}). Demand-side stress is detected earlier in utilization-linked channels. Liquidity-shock signals appear in price, provider-economics, and provider-churn fields around the event window. Cost and yield channels are first detected in provider-economics and provider-churn fields.

The incentive-solvency proxy is used here as a comparative stress signal rather than as a standalone economic score. Conceptually, it asks whether modeled reward outflow and modeled value capture (or usage-linked sinks) remain aligned under stress. Because it behaves like a ratio, it can move sharply when one side deteriorates faster than the other.

Liquidity shock illustrates why the proxy must be read jointly with other fields. Under the same price dislocation, one mechanism may continue distributing rewards while usage-linked sinks weaken quickly, producing a large proxy swing; another may show both sides contracting more tightly together, producing a smaller swing. A larger swing therefore indicates a different transmission path under stress, not a universal ``worse'' outcome by itself.

Accordingly, the proxy bars in Figure~\ref{fig:dtse_cross_profile_retention} are interpreted together with profitability, provider churn, provider retention, and utilization. When these fields move coherently, the proxy helps diagnose reward--demand decoupling; read in isolation, bar height can overstate what the mechanism-level result actually implies.

\subsection{Operational Failure-Mode Signatures}
\label{failure-modes-observed-operational-definitions}
This section defines operational failure-mode signatures as recurring DTSE metric patterns under specified stress inputs. Failure modes are not treated as binary outcomes or protocol-level judgments. They provide a shared vocabulary for describing model-conditional stress responses using pre-specified metrics and thresholds.

For ONO in this thesis calibration, ``emissions'' refer to modeled reward distributions from a fixed token inventory path rather than uncapped token creation. The term is retained for cross-profile comparability because the same metric family is used across mechanism profiles with different supply architectures \cite{OnocoyToken, OnocoyRewards}.

DTSE reporting in this chapter uses two threshold stages. Stage~1 signal-detection thresholds identify earliest-trigger timing fields (Tables~\ref{tab:dtse_signal_timing_ono} and~\ref{tab:dtse_cross_scenario_synthesis}; Appendix Section~\ref{app:dtse_threshold_table}). Stage~2 failure-signature criteria classify recurring stress patterns using the diagnostic matrix in Table~\ref{tab:failure_modes}. The two stages are related but not interchangeable.

\subsubsection{Reward--Demand Decoupling}
Reward--Demand Decoupling occurs when usage-linked activity weakens while reward distributions remain active long enough for the burn-to-mint proxy to deteriorate against baseline. In practical terms, the network keeps paying the supply side while the demand side absorbs less value, increasing the subsidy component inside modeled economics. This is conceptually consistent with burn-credit designs where usage structurally anchors accrual \cite{RenderBME, FrontiersDePIN2025}. In ONO terms, the risk is not new token creation but continued distribution from a fixed inventory under weaker usage, which produces the same solvency pressure in the burn-to-distribution proxy \cite{OnocoyToken, OnocoyRewards}. ONO's reward and bonus policy layer is a concrete example of distribution conditions being adjusted through documented program rules rather than uncapped creation \cite{OnocoyRewards, OnocoyBonus}.

\subsubsection{Profitability-Induced Churn}
Profitability-Induced Churn occurs when provider margins remain negative for sustained windows, even without a one-off market shock. DTSE flags this pattern with the sustained-profitability condition (three consecutive weeks below zero) and rising provider churn relative to baseline. The operational intuition is direct: when operating costs persistently exceed expected reward value, operators power down or reallocate hardware. DePIN participatory-sensing literature treats contributor effort and operating cost coverage as a first-order condition for sustained supply participation \cite{Chiu2024, FrontiersDePIN2025}. Compute-heavy provider settings exhibit similar break-even exit behavior when incentive value compresses \cite{Lin2024}.

\subsubsection{Liquidity-Driven Compression}
Liquidity-Driven Compression occurs when a discrete price dislocation compresses fiat-equivalent reward value before the physical network adjusts. In DTSE, the week-20 shock maps into a price-threshold crossing and records diagnostic detections in profitability and provider churn fields in the same or adjacent windows under frozen rules. The physical layer can appear unchanged in the first window while operator economics move immediately because costs are fiat-like and rewards are token-denominated. A reference example in this thesis is the Helium stress window analyzed in Section~\ref{sec:empirical_analysis}, framed through event-window logic rather than equilibrium assumptions \cite{MacKinlay1997}. The same channel is consistent with evidence that policy and macro uncertainty transmit quickly into crypto-asset stress conditions \cite{Ho2022}.

\subsubsection{Elastic Provider Exit}
Elastic Provider Exit occurs when operators exit under outside-yield pressure before local demand visibly deteriorates. DTSE identifies this pattern when provider churn and active provider count thresholds trigger under competitive-yield pressure while utilization is not the first field to break. Because multi-homing or low-friction reallocation makes supply more mobile than demand, participation can deteriorate before visible service degradation. In GNSS-like settings, ONO and GEODNET are useful reference points for how operators compare incentive conditions across adjacent ecosystems \cite{OnocoyRewards, GeodnetLocationNFT}. GPU-oriented DePIN supply provides a complementary example where re-routing to higher-yield opportunities is structurally easier than rebuilding demand, reinforcing short-horizon exit elasticity \cite{Lin2024, Chiu2024}.

\subsubsection{Latent Capacity Degradation}
Latent Capacity Degradation occurs when participation erodes first while service fields deteriorate later as redundancy is consumed. DTSE records this as a time-order condition where active provider count deviation is detected before utilization/service metrics cross their own thresholds. In plain terms, early operator exits may not break service immediately if overlap remains, but resilience is declining beneath the surface. This lagging pattern is consistent with work on path dependence, hysteresis, and delayed visible effects in complex systems \cite{Arthur1989, Morris2019, Ballandies2023}. Two-sided platform dynamics provide a complementary framing where participation-side erosion can precede visible demand-side deterioration until a threshold is crossed \cite{RochetTirole2003}.

\subsection{Bridge to Discussion}
\label{subsec:deferred_interpretation_register}
Chapter~8 established three discussion anchors under frozen DTSE inputs: first-moving metric families, baseline-relative propagation paths, and recurring failure-mode signatures. Section~\ref{sec:discussion} interprets those anchors in mechanism terms, then maps them to governance-response archetypes and explicit claim boundaries without changing the underlying result set reported here.
