## V2 Writing Behavior Protocol

Status: synthesis control document  
Date captured: 2026-02-28  
Mode: `MODE: SYNTHESIS`

### Purpose

This protocol defines the writing behavior V2 should follow. It is doctrine, not a checklist.

The goal is simple: V2 must not reproduce V1's defensive, over-compressed, mechanically correct prose inside a cleaner shell.

### Core Standard

A section is not acceptable just because it is accurate.

It must also be:

1. readable
2. paced
3. concrete when needed
4. human in tone
5. disciplined in claim handling

If it is academically safe but dead on the page, it fails the V2 standard.

### Drafting Order

#### Old order

1. make it safe
2. make it complete
3. make it structured
4. maybe humanize later

#### V2 order

1. make it land
2. make it legible
3. make it persuasive
4. then enforce precision, citations, and claim boundaries

Rigor is not removed. It is sequenced later.

### Required Drafting Phases

#### 1. Reader Draft

Write for reader understanding first.

Allowed:

- strong transitions
- concrete framing
- examples
- uneven paragraph lengths if they read better

#### 2. Discipline Pass

Tighten:

- claim class
- terminology consistency
- source support
- chapter-contract fit
- duplication
- scope bleed

#### 3. Humanization Pass

Remove:

- repeated stems
- AI-clean cadence
- over-compression
- abstraction overload

#### 4. Submission Pass

Only here do LaTeX, references, figures, and compliance become final.

### Non-Negotiable Rules

1. One paragraph should do one rhetorical job.
2. Difficult ideas should earn abstraction through one concrete grip first.
3. Transitions must connect thoughts, not just headings.
4. Introductions are provisional and should be rewritten later.
5. Dead prose is not preserved just because it is correct.

### Banned Habits

Do not accept sections that rely on:

1. front-loaded scope disclaimers
2. front-loaded interpretive-contract language
3. three or more ideas compressed into one paragraph
4. repeated openings like:
   - `This thesis...`
   - `The thesis...`
   - `This section...`
   - `The goal is...`
   - `The scope is...`
5. subsection symmetry for its own sake
6. defining before motivating
7. qualifying before the idea has landed

### Compression Rule

If a paragraph contains:

1. a definition
2. a why-it-matters claim
3. a caveat
4. and a transition

it is almost certainly overpacked.

Split it.

### Introduction Rule

Introductions are especially vulnerable to defensive academic writing. Therefore:

1. draft them early
2. distrust them early
3. rewrite them after later chapters are stronger

### Model Roles

#### Codex / current model

Responsible for:

- structure
- evidence discipline
- chapter fit
- lexicon control
- final acceptance

#### GPT 5.2

Responsible for:

- humanization
- transitions
- explanation density
- sentence variation
- de-mechanizing prose

Rule:

GPT 5.2 output is draft material until it is checked against chapter contracts, claim boundaries, evidence discipline, and lexicon consistency.

### Red-Flag Examples

#### Bad pattern

`This thesis examines X. The thesis contributes Y. The scope is Z.`

Why it fails:

- structurally neat
- rhetorically flat
- immediately mechanical

Do instead:

- start with the tension
- let the problem land
- introduce contribution and scope later

#### Bad pattern

One paragraph contains definition, caveat, contribution, and transition.

Why it fails:

- efficient on paper
- exhausting in practice

Do instead:

- split by rhetorical job
- let the reader process each move separately

#### Bad pattern

A section opens with formal boundaries before motivation.

Why it fails:

- creates compliance-first reading
- weakens reader pull

Do instead:

- motivate first
- contract second

### Final Rule

Do not preserve dead prose because it is technically correct.

Correctness is necessary. It is not enough.
